## Remaining Issues (14)

### Phase 1: Foundation (4 issues, sessions 1-2)

#### Session 1: CLI + Admin SPA

| Issue | Title | Why first |
|-------|-------|-----------|
| #378 | CLI commands crash with 'no such table: artifact' on fresh DB | Blocks all CLI testing |
| #385 | Admin SPA entity listings hang due to CORS blocking schema/graphql | Blocks manual testing |

**#378 root cause:** Entity types registered in `commands()` may not be available when `getStorage()` is called in the ensureTable loop (line 592). The try-catch silently swallows failures. Fix: ensure all entity types are registered in domain service providers before `commands()` runs.

**#385 root cause:** Admin SPA runs as static build in production. API calls to `/api/schema/` and `/graphql` need Caddy reverse proxy rules or CORS headers. Fix: add Caddy route matcher for `/api/*` or add CORS middleware.

#### Session 2: Deploy + runtime safety

| Issue | Title | Why now |
|-------|-------|---------|
| #372 | Ansible-managed .env overwrites manual secret changes on deploy | Deploy safety |
| #326 | Daily ceiling check in InternalSessionController is not atomic | Runtime safety |

**#372 note:** This is a cross-repo issue (northcloud-ansible). deploy.php already has `.env` in `shared_files`. The issue may be about Caddyfile template overwrites, not .env directly.

**#326 root cause:** InternalSessionController:68-90 loads all tenant sessions, filters by date in PHP, sums turns. TOCTOU race on concurrent requests. Comment says "acceptable given single-user tenants." Fix: add database-level locking or accept the documented trade-off and close.
