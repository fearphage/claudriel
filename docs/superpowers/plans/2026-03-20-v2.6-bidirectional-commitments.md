**Goal:** Extend Claudriel's commitment extraction to detect inbound commitments (what others owe you), monitor unanswered emails, detect completion from replies, and surface a "Waiting On" section in the daily brief.

**Architecture:** Add a `direction` field to Commitment (`outbound` = you owe them, `inbound` = they owe you). Modify the AI extraction prompt to classify direction. Add a FollowUpMonitor service that detects unanswered sent emails. Add a CompletionDetector that marks inbound commitments done when fulfilled. Extend DayBriefAssembler with a "Waiting On" section.

**Tech Stack:** PHP 8.4, Waaseyaa AI Pipeline (Anthropic API), SQLite entity storage, Twig templates, Pest/PHPUnit tests.

```php
final class FollowUpMonitor
{
    private const DEFAULT_DAYS_THRESHOLD = 3;

    public function __construct(
        private readonly EntityTypeManager $entityTypeManager,
    ) {}

    /**
     * Find sent emails older than $daysThreshold that have no reply in their thread.
     *
     * @return list<array{thread_id: string, subject: string, sent_at: string, recipient: string}>
     */
    public function findUnanswered(string $tenantId, int $daysThreshold = self::DEFAULT_DAYS_THRESHOLD): array
    {
        $storage = $this->entityTypeManager->getStorage('mc_event');
        $cutoff = (new \DateTimeImmutable("-{$daysThreshold} days"))->format(\DateTimeInterface::ATOM);

        // Load sent emails older than threshold
        $sentIds = $storage->getQuery()
            ->condition('type', 'message.sent')
            ->condition('tenant_id', $tenantId)
            ->execute();

        $sent = $sentIds !== [] ? $storage->loadMultiple($sentIds) : [];

        // Load all received emails (potential replies)
        $receivedIds = $storage->getQuery()
            ->condition('type', 'message.received')
            ->condition('tenant_id', $tenantId)
            ->execute();

        $received = $receivedIds !== [] ? $storage->loadMultiple($receivedIds) : [];

        // Build set of thread IDs that have replies
        $repliedThreads = [];
        foreach ($received as $event) {
            $payload = json_decode($event->get('payload') ?? '{}', true);
            $threadId = $payload['thread_id'] ?? null;
            if ($threadId !== null) {
                $repliedThreads[$threadId] = true;
            }
        }

        // Filter sent emails: older than cutoff + no reply in thread
        $unanswered = [];
        foreach ($sent as $event) {
            $payload = json_decode($event->get('payload') ?? '{}', true);
            $threadId = $payload['thread_id'] ?? null;
            $sentAt = $event->get('occurred') ?? '';

            if ($threadId === null || $sentAt > $cutoff) {
                continue;
            }
            if (isset($repliedThreads[$threadId])) {
                continue;
            }

            $unanswered[] = [
                'thread_id' => $threadId,
                'subject' => $payload['subject'] ?? '(no subject)',
                'sent_at' => $sentAt,
                'recipient' => $payload['to_email'] ?? $payload['to'] ?? '',
            ];
        }

        return $unanswered;
    }
}
```

- [ ] **Step 3: Implement CommitmentCompletionDetector**

The detector finds inbound commitments where the source thread has received a new message since the commitment was created. This suggests the person followed through.

```php
final class CommitmentCompletionDetector
{
    public function __construct(
        private readonly EntityTypeManager $entityTypeManager,
    ) {}

    /**
     * Find inbound commitments whose source thread received a follow-up message.
     *
     * @return list<ContentEntityInterface> Commitments that appear completed
     */
    public function detectCompleted(string $tenantId): array
    {
        $commitmentStorage = $this->entityTypeManager->getStorage('commitment');
        $eventStorage = $this->entityTypeManager->getStorage('mc_event');

        // Load pending inbound commitments
        $ids = $commitmentStorage->getQuery()
            ->condition('direction', 'inbound')
            ->condition('status', 'pending')
            ->condition('tenant_id', $tenantId)
            ->execute();

        $commitments = $ids !== [] ? $commitmentStorage->loadMultiple($ids) : [];

        // For each, check if the source thread has a newer received message
        $completed = [];
        foreach ($commitments as $commitment) {
            $sourceEventId = $commitment->get('source_event_id');
            if ($sourceEventId === null) {
                continue;
            }

            $sourceEvent = $eventStorage->load($sourceEventId);
            if ($sourceEvent === null) {
                continue;
            }

            $sourcePayload = json_decode($sourceEvent->get('payload') ?? '{}', true);
            $threadId = $sourcePayload['thread_id'] ?? null;
            if ($threadId === null) {
                continue;
            }

            // Check for newer received messages in this thread
            $replyIds = $eventStorage->getQuery()
                ->condition('type', 'message.received')
                ->condition('tenant_id', $tenantId)
                ->execute();

            foreach ($replyIds !== [] ? $eventStorage->loadMultiple($replyIds) : [] as $reply) {
                $replyPayload = json_decode($reply->get('payload') ?? '{}', true);
                if (($replyPayload['thread_id'] ?? null) === $threadId
                    && ($reply->get('occurred') ?? '') > ($sourceEvent->get('occurred') ?? '')) {
                    $completed[] = $commitment;
                    break;
                }
            }
        }

        return $completed;
    }
}
```
